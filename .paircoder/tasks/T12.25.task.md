# Objective

Create `backend/scripts/nightly_digest.py` as the APScheduler job handler. Runs at 02:00 city-local (via T12.3 day-boundary helper). S12a scope — plan-refresh (T12.24) and reminder-engine-with-cooldown (T12.19) land in S12b. For each active session: (1) run retro for yesterday, (2) check stall signals, (3) compose digest for today, (4) send via T12.2 SendGrid directly. Orchestrator iterates cities sequentially; each city's sessions processed in a city-scoped batch.

# Implementation Plan

1. Implement script iterating active sessions per-city with `asyncio.Semaphore(10)` within each city.
2. Enforce city scope — sessions from City A never processed in City B invocation.
3. Wrap each session in try/except so single-session errors don't abort the batch.
4. Port `ops:lib/nightly_accounting.py` (`build_accounting_record`, `save_session`, `load_session`) — write accounting record per run to `nightly_run_log` with `city, sessions_processed, emails_sent, errors, duration_sec, start_ts, end_ts`.
5. Handle empty-session-pool gracefully.
6. Register job with APScheduler; gate on `FEATURE_NIGHTLY_ENABLED` (kill switch).
7. Stub plan-refresh step with TODO pointing to S12b T12.24.
8. Direct SendGrid send via T12.2; TODO for S12b T12.19 replacement.
9. Write tests: empty pool, single-session success, multi-session with failure, kill switch, city-scope isolation.
