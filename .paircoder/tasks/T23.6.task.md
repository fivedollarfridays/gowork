# Objective

`GET /api/assessments/{slug}` returns the latest published version + questions. Anonymous-friendly (the route MUST work for both anonymous and account-claimed sessions per the integrity charter). Drafts are never publicly fetchable — only `status=published` versions appear. Cacheable for 60s.

Lands in a NEW route module — must NOT extend `backend/app/routes/auth.py`.

# Files

- `backend/app/routes/assessments_public.py` (new)
- `backend/app/routes/__init__.py` (register new router)
- `backend/tests/test_assessments_public.py` (new)

# Acceptance Criteria

- [x] `GET /api/assessments/{slug}` returns the latest published version + questions
- [x] No auth required; the route is fully public
- [x] Anonymous-first invariant test (`tests/test_anonymous_first_invariant.py`) auto-discovers this route via FastAPI route inspection and asserts equivalence between anonymous and claimed sessions
- [x] Response schema: `{slug, kind, track, version_number, published_at, questions: [{position, prompt, kind, scoring_weight}]}`
- [x] **`rubric_json` EXCLUDED from public response** (rubrics are reviewer-internal; leaking them would let candidates game the scoring)
- [x] 404 if no published version exists for the slug (drafts are never publicly fetchable)
- [x] Sets `Cache-Control: public, max-age=60`
- [x] `assessments_public_router` registered in `backend/app/routes/__init__.py:all_routers`
- [x] Tests cover: published-200, draft-only-404, anon-vs-claimed-equivalence, rubric-not-leaked, cache-header
- [x] `bpsai-pair arch check` passes
