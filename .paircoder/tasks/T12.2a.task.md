# Objective

SendGrid sends bounce and spam-complaint notifications to a webhook. Without ingestion, reminders degrade silently in prod and reputation damage accrues. Create `backend/app/routes/sendgrid_webhook.py` to receive `POST /api/webhooks/sendgrid/events` with HMAC-signed payload verification. Writes to `engagement_events(category='bounce'|'spam_report'|'delivered'|'open')` and `sendgrid_events` table. On **hard bounce**, auto-sets `reminders_enabled=False` on the affected session.

# Implementation Plan

1. Create `POST /api/webhooks/sendgrid/events` verifying SendGrid signature via `SENDGRID_WEBHOOK_PUBLIC_KEY` (ECDSA per SendGrid spec).
2. Handle event types: `bounce`, `dropped`, `spam_report`, `delivered`, `open`.
3. On hard bounce: set `sessions.reminders_enabled = False` + audit entry.
4. On soft bounce: log-only, no auto-action.
5. On spam complaint: set `sessions.reminders_enabled = False` + advisor flag.
6. Mirror all events to `engagement_events` (category) AND `sendgrid_events` (raw payload).
7. Return 401 on unsigned / malformed payloads; log.
8. Register router in `all_routers`.
9. Write tests: signature verify happy + fail, each event type, hard-bounce side effect.
