# Objective

`POST /api/admin/assessments/draft` accepts a slug + kind + track + source_prompt. Calls Claude via the existing `app.ai.client` to generate questions matching the assessment kind. Validates the structured response (Pydantic schema), persists the assessment + draft version, returns the version_id. Rate-limited per account. Mock provider used in tests for determinism.

Lands in a NEW route module — must NOT extend `backend/app/routes/auth.py` (already at 301 lines).

# Acceptance Criteria

- [ ] `POST /api/admin/assessments/draft` accepts `{slug, kind, track, source_prompt}`
- [ ] Response: `{assessment_id, version_id, status: "draft", question_count}`
- [ ] Gated: `require_role("admin")` OR `require_role("sme_reviewer")` (drafters); 403 otherwise
- [ ] `app/ai/assessment_drafter.py` exposes `draft_questions(kind, track, source_prompt) -> list[QuestionDraft]` calling existing `app.ai.client` via `prompt_router`
- [ ] Pydantic `QuestionDraft` validates `{prompt, kind, rubric, scoring_weight}` from Claude output; malformed → 502 with sanitized error
- [ ] Rate limit: 10/hour per account (reuses `app.core.rate_limit.RateLimiter`)
- [ ] `Settings.assessment_drafter_model` defaults to `claude-sonnet-4-6`; configurable
- [ ] `assessments_admin_router` registered in `backend/app/routes/__init__.py:all_routers`
- [ ] Tests use mock provider for determinism: happy path, rate-limit, malformed Claude response, role enforcement (admin OK, sme_reviewer OK, case_manager 403, anonymous 403)
- [ ] Live Claude call deferred to manual verification
- [ ] `bpsai-pair arch check` passes on all new files
