# Implementation Plan

1. Define `Appointment` Pydantic model with all fields: id, session_id, type, title, starts_at, ends_at, location_name, location_address, barrier_link, status, source, notes.
2. Import `AppointmentType`/`AppointmentStatus` enums from T12.5 `temporal_types`.
3. Make `barrier_link` optional FK to `barrier_id` for pathway-sourced appointments.
4. Ensure JSON-serializable for API responses.
5. Add validators for past dates, zero-duration (ends_at <= starts_at), missing location.
6. Write tests covering model validation edge cases.

# Acceptance Criteria
