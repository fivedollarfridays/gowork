# Implementation Plan

1. Add `apscheduler` to `backend/requirements.txt`.
2. Wire APScheduler startup/shutdown into FastAPI lifespan; handle SIGTERM cleanly.
3. Expose `register_job(name, func, trigger)` helper.
4. Register `nightly_digest` (02:00 city-local), `stall_scan` (08:00 daily), `appointment_reminders` stub.
5. Use city config timezone; port `current_work_date`, `resolve_work_date`, `_resolve_rollover_hour` from `ops:lib/nightly_day_boundary.py` to `backend/app/core/day_boundary.py`.
6. Enforce multi-worker safety via `scheduler_leases` atomic lease OR startup check for `WEB_CONCURRENCY=1`.
7. Guard HTTP-triggered job runs behind `require_admin_key`.
8. Write tests for registration, multi-worker lock under `WEB_CONCURRENCY=2`, day-boundary across DST + rollover hours.
