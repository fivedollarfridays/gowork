# Objective

`POST /api/admin/assessments/{version_id}/publish` triggered ONLY when status=approved. Admin-only (publish authority is narrower than review). Sets `published_at`, status=published, locks the version (subsequent edits return 409). Returns the public URL the candidate-side will eventually consume.

Provenance is the load-bearing charter principle here: every published row must have drafted_by + reviewed_by + approved_by + published_at all populated.

# Files

- `backend/app/routes/assessments_review.py` (extend)
- `backend/tests/test_assessments_publish.py` (new)

# Acceptance Criteria

- [x] `POST /api/admin/assessments/{version_id}/publish` extends the review router
- [x] Gated: `require_role("admin")` — strict admin tier (sme_reviewer cannot publish)
- [x] Guards: status must be approved (409 with explicit reason if not); already-published returns 409 (no double-publish)
- [x] Sets `published_at` (utcnow) + status=published via `queries_assessments.publish_version`
- [x] Returns `{assessment_id, version_id, slug, version_number, published_at, public_url}` where `public_url = f"/api/assessments/{slug}"`
- [x] Provenance assertion at the test level: drafted_by, reviewed_by, approved_by, published_at all non-null on the published row
- [x] Subsequent calls to `record_review` on a published version → 409
- [x] Tests cover: approve-then-publish happy path, publish-without-approve-409, double-publish-409, role-enforcement (sme_reviewer 403, case_manager 403, anonymous 403), provenance row inspection
