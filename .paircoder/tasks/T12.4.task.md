# Objective

Add WeasyPrint to `backend/requirements.txt`. Create `backend/app/core/pdf_renderer.py` with `render_markdown_to_pdf(markdown_content, template_name)`. Templates live in `backend/app/modules/documents/templates/`. Update Dockerfile. SSRF defense: deny-all `url_fetcher` stub so external URL references in HTML (e.g. `<img src="http://169.254.169.254/">`) cannot be fetched by the renderer.

# Implementation Plan

1. Add WeasyPrint to `backend/requirements.txt`; pin the version.
2. Create `pdf_renderer.py` with Jinja2 `Environment(autoescape=True)`; pre-escape worker strings with `html.escape()`.
3. Implement SSRF-safe `url_fetcher` that raises `URLFetchingError` for all external URLs.
4. Render markdown to HTML, then HTML to PDF bytes; verify magic bytes + page count in tests.
5. Update Dockerfile with `libpango-1.0-0 libpangoft2-1.0-0 libcairo2 fonts-liberation` under `--no-install-recommends`; keep `python:3.13-slim` + non-root `appuser`.
6. Document image-size delta in the PR description.
7. Test with injected `<img src="http://169.254.169.254/">` and assert blocked.

# Acceptance Criteria

- [ ] `render_markdown_to_pdf()` accepts markdown string + template name, returns bytes
- [ ] WeasyPrint instantiated with `url_fetcher` stub raising `URLFetchingError` for all external URLs (test injects `<img src="http://169.254.169.254/">` and asserts blocked)
- [ ] Jinja2 `Environment(autoescape=True)`; all user-supplied string fields pre-escaped via `html.escape()` before interpolation
- [ ] Dockerfile: `RUN apt-get install -y --no-install-recommends libpango-1.0-0 libpangoft2-1.0-0 libcairo2 fonts-liberation` (compatible with `python:3.13-slim` + non-root `appuser`)
- [ ] Docker image size delta documented in PR
- [ ] Test renders sample markdown to valid PDF (verify magic bytes + page count)
- [ ] `bpsai-pair arch check` passes
