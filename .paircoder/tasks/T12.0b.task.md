# Objective

`ENABLE_AI_GENERATION` (S12b-only), `FEATURE_NIGHTLY_ENABLED` (default **true**), and `FEATURE_EMAIL_SEND_ENABLED` (default **true**) are load-bearing toggles. Create `backend/app/core/feature_flags.py` with stdlib-only resolution chain (env → yaml → default) and a `require_admin_key`-protected admin endpoint for runtime toggle with structured audit.

# Implementation Plan

1. Create `feature_flags.py` with `is_enabled(flag_name, default=False) -> bool` reading env `FEATURE_<NAME>`, then `config/feature_flags.yaml`, then default.
2. Create `POST /api/admin/flags/{name}` endpoint with body `{enabled: bool, reason: str}` protected by `require_admin_key`.
3. Write audit record to `feature_flag_audit` (m002, owned by T12.1): `flag_name, old_value, new_value, reason, actor_token_hash (SHA256), source_ip, timestamp`.
4. Implement rate limit: max 10 toggle ops per admin token per hour (429 on exceed).
5. Log explicit warning when `ENABLE_AI_GENERATION` enabled: "AI generation enables LLM processing of worker PII — verify data processing agreement is active".
6. Document defaults in `.env.example`.
7. Write tests: resolution chain, admin toggle, audit write with all fields, rate-limit 429, AI warning emission.

# Acceptance Criteria
