# Objective

`POST /api/employers/claim {listing_id, claimant_email}` accepts an email, mints a single-use token, sends a magic-link-style claim URL via SendGrid. Always returns 202 (no enumeration). Tighter rate limits than candidate magic-link since this is employer-side. Domain heuristic: extract domain from `claimant_email`; if it fuzzy-matches `job_listings.company`, route to claim_verified tier on subsequent verify; otherwise route to admin_review queue.

Lands in NEW route module — must NOT extend `auth.py` (314 lines, 400 ceiling).

# Acceptance Criteria

- [ ] `POST /api/employers/claim` accepts `{listing_id: int, claimant_email: str}`; response 202 always
- [ ] Token: `secrets.token_urlsafe(32)` ≥256-bit entropy; SHA-256 hashed
- [ ] 15-min expiry; single-use
- [ ] Domain heuristic: extract domain from claimant_email; lookup or auto-create employer_accounts row keyed by domain
- [ ] If `job_listings.company` doesn't fuzzy-match the domain → mark prospective claim with target tier=admin_review
- [ ] SendGrid email sent with claim URL `{FRONTEND_URL}/employers/claim?token={token}`
- [ ] Rate limits: 5/hour per IP, 3/hour per email (return 202 anyway when limited)
- [ ] Router registered in all_routers
- [ ] AUDIT_ALLOWLIST + PUBLIC_ENDPOINTS extended
- [ ] Tests cover: happy path domain-match, domain-mismatch routes to admin queue, rate-limit, no-enumeration, invalid email → 422
