# Objective

Extend `frontend/src/app/case-manager/page.tsx` with a "Needs Attention" section listing stalled sessions. Advisor can drill through and send a personal note via the reminder engine. City-scoped queries; cross-city returns 403. Prerequisite: `docs/security/advisor-auth.md` (written in T12.31a) must exist before this task starts.

# Implementation Plan

1. Implement advisor auth per T12.31a doc: per-advisor tokens with `city` claim.
2. Backend: `/api/advisor/stalled-sessions` filters by `city = advisor.city`; cross-city 403.
3. Frontend: list stalled sessions with severity indicator + days stalled.
4. Click-through → session detail view (scoped).
5. "Send note" calls reminder engine with custom message; rate-limited per advisor.
6. Sort by severity then days-stalled.
7. Tests cover list, detail drilldown, send-note, cross-city 403.
