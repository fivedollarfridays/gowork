# Objective

Author `data/cities/dallas/community_resources.json`, `data/cities/dallas/resources.json`, and `data/cities/dallas/career_centers.json` with curated Dallas-area entries from open sources. Schema parity with the FW JSON shape — every entry includes `name`, `category`, `subcategory`, `address`, `lat`, `lng`, `phone`, `url`. All coordinates geocoded (no `0.0` placeholders). Eligibility entries for any new resource names introduced via T25.1's `DALLAS_ELIGIBILITY_RULES` are confirmed present.

Loaders are city-agnostic — the existing FW resource loader points at `data_dir` from the city config. Schema drift surfaces as test failures in T25.6 (city-aware router validation), so this task's correctness is enforced downstream.
