# Objective

Port `ops:lib/schedule.py` (`load_schedule_blocks`, `add_schedule_block`, `get_unavailable_blocks`, `add_event`) to `backend/app/modules/appointments/unavailability.py`. The `worker_unavailability` table already exists in the S12a m002 migration; this task fills in the persistence + read API. T12.8 (availability engine) will subtract these blocks from available slots.

# Implementation Plan

1. Implement `add_unavailability_block(session_id, day_of_week, start_time, end_time, reason)` writing to `worker_unavailability`.
2. Implement `get_unavailable_blocks(session_id)` returning rows for availability subtraction.
3. Implement `list_schedule_events(session_id, date_range)` for UI display.
4. Ensure T12.8 `compute_slots` consumes `get_unavailable_blocks` (integration verified in T12.8 tests).
5. Tests cover block creation, overlap with existing appointments, and subtraction semantics.
