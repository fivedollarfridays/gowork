# Objective

Implement `get_signal_rates(listing_id, window_days=30)` returning `{response_rate, withdrawal_rate, placement_rate, sample_size}`. Rolling 30-day window, computed on demand from listing_reputation_events (no denormalization yet — keep schema simple; future sprint can materialize a rolling window if perf demands).

The only P1 task this sprint — substrate works without computed rates; this is read-side ergonomics. **Cuttable scope.**
