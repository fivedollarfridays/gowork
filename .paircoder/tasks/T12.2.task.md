# Objective

Add `backend/app/integrations/email/` module with thin SendGrid wrapper. `send_transactional(to, subject, html, text_fallback, category)`. Retry logic (3 attempts, exponential backoff). Respects `FEATURE_EMAIL_SEND_ENABLED` kill switch. Failed sends logged to `engagement_events`.

# Implementation Plan

1. Create `send_transactional(to, subject, html, text_fallback, category)` with retry (3 attempts, exponential backoff) and structured logging.
2. Support category tagging: digest, reminder, stall-alert, appointment-confirmation, appointment-reminder.
3. Implement mock provider for tests (no live API calls).
4. Gate on `FEATURE_EMAIL_SEND_ENABLED` from T12.0b — when false, no-op + audit log.
5. Log failed sends to `engagement_events` with error detail.
6. Pre-send dedup check: skip address if recent hard-bounce event present (populated by T12.2a).
7. Document `SENDGRID_API_KEY` in `.env.example` scoped to **Mail Send only** (no contact list, sender management, activity read).
8. Write runbook at `docs/ops/sendgrid-setup.md`: sender domain auth (SPF/DKIM), scoping.
9. Write tests: retry/backoff, structured log emission, failure path, kill switch, bounce dedup.
