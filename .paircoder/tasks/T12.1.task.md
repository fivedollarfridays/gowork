# Objective

Create `m002_s12_worker_companion.py` (via T12.0's module convention) adding **all** new tables required across S12a AND S12b — schema-first means every table exists before any module reads it. All `session_id` columns are `TEXT` (matches `sessions.id TEXT PRIMARY KEY`). All session FKs declare `ON DELETE CASCADE`. `plan_history` coexists with existing `sessions.previous_plan` column — dual-write during transition; deprecation scheduled for S13.

# Implementation Plan

1. Create `m002_s12_worker_companion.py` with `SCHEMA_VERSION = 2`, `upgrade(conn)`, `downgrade(conn)`.
2. Define all 13 tables: `appointments`, `job_applications`, `resume_versions` (S12b), `daily_progress_snapshots`, `engagement_events`, `plan_history` (S12b), `outcomes_records`, `reminder_cooldowns` (S12b), `nightly_run_log`, `scheduler_leases`, `worker_unavailability` (S12b), `feature_flag_audit`, `sendgrid_events`.
3. All `session_id` columns TEXT with `REFERENCES sessions(id) ON DELETE CASCADE`.
4. Add indexes on `session_id`, `status`, `starts_at`, `applied_date`, `date`, `created_at`.
5. `plan_history` schema: `id, session_id, archived_at, plan_json, refresh_reason, triggering_event` (cap of 20 per session enforced in T12.24; add comment).
6. Ensure idempotency and test `downgrade` round-trip on seeded DB.
7. Write test that greps the migration source and asserts all 13 tables present.
