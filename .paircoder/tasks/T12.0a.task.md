# Objective

Replace the in-memory dict upsert in `backend/app/modules/outcomes/tracker.py:24-26` with a SQLite-backed append-only store. Current implementation overwrites prior outcomes and loses all state on restart — this breaks the "N+1 friendly" principle and is assumed as an append-only signal sink by T12.7, T12.11, T12.18, T12.22, T12.23. Reference patterns: `ops:lib/conversion_tracking.log_conversion` and `ops:lib/block_decisions.record_decision` (both append-only). All 5 existing callers must be updated in this same task.

# Implementation Plan

1. After T12.1 creates `outcomes_records` table, rewrite `OutcomeTracker` with `OutcomeTracker(db_path)` constructor.
2. Implement `record_outcome(record)` as INSERT (never upsert).
3. Add `list_by_session(session_id) -> list[OutcomeRecord]` returning chronological history.
4. Add `list_recent(city, event_type, since)` for aggregate intelligence queries.
5. Preserve `get_latest(session_id)` convenience method for `intelligence.py` pattern; document audit of call sites in PR.
6. Update all 5 callers: `aggregator.py`, `intelligence.py`, `test_outcome_tracker.py`, `test_outcome_aggregator.py`, `test_s8_coverage_gaps.py`.
7. Write tests: multiple writes per session, process-restart durability, cross-session aggregates, CASCADE on session delete.
