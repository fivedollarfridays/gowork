# Objective

Create `backend/app/modules/plan/plan_refresher.py`. When retro detects HARD stall or breakthrough, `refresh_plan(session_id, trigger_reason)` re-runs the pathway engine with adjusted inputs, archives the old plan to `plan_history` (S12a m002), and carries forward checklist state via ported `ops:lib/plan_progress.py` (`load_existing_progress`, `merge_existing_progress`, `_extract_progress`). Dual-writes to existing `sessions.previous_plan` column during S12b (deprecation scheduled for S13). Enforces the 20-row-per-session cap the migration comment promised.

# Implementation Plan

1. Port `plan_progress.py` — `load_existing_progress`, `merge_existing_progress`, `_extract_progress`.
2. Implement `refresh_plan(session_id, trigger_reason)`: re-run pathway with calibrated_weeks from intelligence engine.
3. Archive old plan to `plan_history` with refresh_reason + triggering_event.
4. Dual-write to `sessions.previous_plan` column (note deprecation in S13).
5. Enforce 20-row cap: after archive, delete rows older than top-20 by session.
6. Carry checklist state via `merge_existing_progress(old_plan, new_plan)`.
7. Wire into T12.25 orchestrator step 3; remove the TODO stub.
8. Tests: stall-triggered, breakthrough-triggered, no-trigger, progress carry-forward, history cap enforcement.
