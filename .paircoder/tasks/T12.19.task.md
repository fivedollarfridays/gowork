# Objective

Create `backend/app/modules/engagement/reminder_engine.py`. Dispatches email based on stall level (SOFT/MEDIUM/HARD templates). Port `ops:lib/cooldown.py` (`check_cooldown`, `get_cooldown_status`, `_apply_cooldown_filter`) to `backend/app/modules/engagement/cooldown.py`. Backed by `reminder_cooldowns` table (S12a m002). Dedup keyed on `(session_id, stall_level)` — NOT `(session_id, barrier_id)` — so a multi-barrier stalled session gets exactly one SOFT email per 24h, not one per barrier.

# Implementation Plan

1. Port `cooldown.py` — `check_cooldown`, `get_cooldown_status`, `_apply_cooldown_filter`.
2. Build 3 templates (SOFT/MEDIUM/HARD) with HTML-escaped worker fields.
3. Implement dedup keyed on `(session_id, stall_level)` via `reminder_cooldowns`.
4. Include unsubscribe link routing to T12.21 endpoint (CAN-SPAM compliance).
5. Respect `reminders_enabled=False` from worker opt-out or T12.2a hard-bounce handler.
6. Update T12.25 orchestrator to call `reminder_engine.send_digest` instead of direct SendGrid step.
7. Write test: 3-barrier stalled session produces exactly 1 SOFT email per 24h.
