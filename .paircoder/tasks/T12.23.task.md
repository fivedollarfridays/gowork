# Objective

Create `backend/app/modules/plan/evidence_collector.py`. `collect_evidence(session_id, date_range)` returns unified `EvidenceBundle` (checklist_items_completed, appointments_attended, appointments_missed, applications_filed, applications_progressed, outcomes_logged). Moved ahead of T12.22 retro and T12.20 digest which both depend on it.

# Implementation Plan

1. Define `EvidenceBundle` Pydantic model with all six signal types.
2. Implement `collect_evidence(session_id, date_range)` reading from appointments (T12.7), job applications (T12.11), and outcomes store (T12.0a).
3. Make date-range inclusive on both ends.
4. Enforce city scope (no cross-city) via session's city.
5. Read historical events from T12.0a DB-backed outcomes store.
6. Write tests covering overlapping ranges, empty range, single-day, multi-session isolation.
