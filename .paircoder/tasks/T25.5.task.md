# Objective

Ship `scripts/import_gtfs.py` — a reusable GTFS-to-FW-JSON-shape importer. Accepts a GTFS zip path + city slug; emits `transit_routes.json` + `transit_stops.json` matching the canonical FW shape. Routes carry `{route_number, route_name, weekday_start, weekday_end, saturday, sunday}`; stops carry `{route_id, stop_name, lat, lng, sequence}`. Run the importer against the public DART GTFS feed; commit the output under `data/cities/dallas/`. Ship a runbook so future cities (Houston METRO, etc.) get the same path for free.

**Spotlight invention (sprint level):** the importer is the contract boundary between any GTFS feed and the seed format. Schema drift surfaces here, not in downstream consumers — and the importer is the multi-city force multiplier called out in the brief.

# Acceptance Criteria

- [x] `scripts/import_gtfs.py` accepts `--gtfs-zip <path>` and `--city <slug>` flags; reads the zip via stdlib `zipfile` (no third-party GTFS lib required)
- [x] Outputs `data/cities/<slug>/transit_routes.json` matching shape: `[{route_number: int, route_name: str, weekday_start: "HH:MM", weekday_end: "HH:MM", saturday: bool, sunday: bool}, ...]`
- [x] Outputs `data/cities/<slug>/transit_stops.json` matching shape: `[{route_id: int, stop_name: str, lat: float, lng: float, sequence: int}, ...]`
- [x] `weekday_start`/`weekday_end` derived from `stop_times.txt` MIN/MAX over the `service_id` whose `calendar.txt` row has any of `monday`-`friday` set
- [x] `saturday`/`sunday` derived from `calendar.txt` boolean flags for the route's primary service_id
- [x] Stop sequence preserves GTFS `stop_sequence` per route
- [x] Idempotent: re-running on the same GTFS produces byte-identical JSON (sorted keys via `json.dumps(..., sort_keys=True)`, stable iteration, deterministic floats — round to 4 decimals)
- [x] `scripts/__tests__/test_import_gtfs.py` feeds the fixture mini-GTFS (3 routes, 8 stops) and asserts: route count, stop count, schema match, weekday boundary derivation, sat/sun flag mapping, idempotency byte-match across two runs
- [x] Dallas DART import: `python scripts/import_gtfs.py --gtfs-zip <DART_GTFS> --city dallas` produces non-empty seed files committed under `data/cities/dallas/`
- [x] DART output sanity floors: ≥80 routes, ≥1000 stops (DART operates ~120 routes; floors catch a broken service_id filter)
- [x] `docs/import-gtfs-runbook.md` documents: where to fetch each city's GTFS feed, license/attribution requirements, importer invocation, expected output shape, sanity floors per city
- [x] `bpsai-pair arch check scripts/import_gtfs.py` passes
- [x] Importer module under 400 lines (architecture constraint)
