# Objective

Create `backend/app/modules/engagement/stall_detector.py`. `scan_active_sessions()` computes `days_since_last_progress` per session. Returns `StalledSession(session_id, stalled_barriers, days_stalled, stall_level)`. SOFT (≥3d), MEDIUM (≥7d), HARD (≥14d). Port `ops:lib/engagement_status.py` (`get_engagement_status`, `_get_recommendations`). S12a scope: detect and classify only; reminders land in T12.19 (S12b). Auto-advanced appointments (S12b T12.25a) filtered: `event_type='appointment_auto_advance'` within 48h does NOT count as stall signal.

# Implementation Plan

1. Implement `scan_active_sessions()` reading from T12.23 evidence collector.
2. Compute `days_since_last_progress` per session; classify SOFT/MEDIUM/HARD by thresholds.
3. Track stall per-barrier (not just session-level).
4. Implement `get_engagement_status(session_id)` returning enriched struct with `_get_recommendations` output.
5. Filter outcomes with `event_type='appointment_auto_advance'` within last 48h from stall signal.
6. Ensure idempotency — running twice same day produces same result.
7. Write tests: all stall levels, no-stall, multi-barrier mixed, auto-advance suppression.
