# Objective

Ship `POST /auth/magic-link` that accepts an email, mints a single-use opaque token (≥256-bit entropy), persists it in `account_credentials` with a 15-minute expiry, and emails it via the existing SendGrid integration. Always returns `202 Accepted` regardless of whether the email exists (no enumeration). Rate-limited per email and per IP.

# Files

- `backend/app/routes/auth.py` (new)
- `backend/app/core/queries_accounts.py` (extend with credential mint helpers)
- `backend/tests/test_auth_magic_link.py` (new)

# Acceptance Criteria

- [x] `backend/app/routes/auth.py` exposes `POST /auth/magic-link`
- [x] Request body: `{email: str}`; response: `202 Accepted` (always, no body)
- [x] Token: ≥256-bit entropy via `secrets.token_urlsafe(32)`; stored hashed (SHA-256) in `credential_value_hash`
- [x] Token expires 15 minutes from issuance
- [x] If the email maps to no account, create one transparently (account-on-first-use)
- [x] SendGrid email sent with claim URL: `{FRONTEND_URL}/auth/claim?token={token}`
- [x] Rate limits: 3/hour per email, 10/hour per IP (return 202 anyway when limited; log)
- [x] `backend/tests/test_auth_magic_link.py` covers issuance, no-enumeration, expiry, rate-limit
- [x] `bpsai-pair arch check` passes
