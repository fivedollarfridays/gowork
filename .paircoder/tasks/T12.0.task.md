# Objective

Create a lightweight migration system for `backend/app/core/`. The current `schema.py` ships as a single `DDL_SQL` blob with no versioning. Follow the `ops:lib/db.py` pattern (`_ensure_schema`, `_current_version`, `_apply_migrations`) with per-migration Python modules (`m001_initial.py`, `m002_s12_worker_companion.py`, ...) each exporting `SCHEMA_VERSION = N` and `upgrade(conn)` / `downgrade(conn)` functions.

# Implementation Plan

1. Create `backend/app/core/migrations/` package with `runner.py` exposing `apply_pending(db_path)`.
2. Auto-create `schema_migrations(version INTEGER PRIMARY KEY, applied_at TEXT NOT NULL)` tracking table on first run.
3. Extract `DDL_SQL` verbatim into `m001_initial.py` with `SCHEMA_VERSION = 1`, `upgrade(conn)`, `downgrade(conn)`.
4. Implement discovery (glob `m*.py`), ordering by SCHEMA_VERSION, and forward-only application.
5. Add `--rollback <version>` flag calling the module's `downgrade(conn)` function.
6. Add dry-run mode that prints SQL without executing.
7. Write tests: fresh DB, existing DB with zero migrations, re-run idempotency, rollback, schema_migrations integrity.
