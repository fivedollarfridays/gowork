# Objective

Create `backend/app/routes/documents.py` with 7 endpoints (resume/cover-letter POST/GET/GET-pdf + versions). Uses T12.4's SSRF-guarded PDF renderer. Explicit version-use-counter hook wired from `applications.py:create()`.

# Implementation Plan

1. Design 7 endpoints: POST resume, GET resume, GET resume PDF, POST cover letter, GET cover letter, GET cover letter PDF, GET versions list.
2. Ensure content-type correctness: `text/markdown` for markdown, `application/pdf` for PDF.
3. Newest-first ordering on version history.
4. Wire `applications.py:create()` to call `resume_versions.increment_use_counter(version_id)`; document hook.
5. Register router in `routes/__init__.py` alongside other S12 routers.
6. Write endpoint tests + PDF byte-content validation.
