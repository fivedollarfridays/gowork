# Objective

Fills the `appointment_reminders` scheduler job stub from S12a T12.3. Port `ops:lib/booking_emails.py` (`send_confirmation`, `send_reminder_24h`, `send_reminder_1h`, `build_manage_url`, `_render_email`) to `backend/app/modules/appointments/transactional_emails.py`. Subscribes to S12a event bus on `appointment.created` to send confirmation. The `appointment_reminders` scheduler job (every 6h) scans upcoming appointments in the 24h and 1h windows.

# Implementation Plan

1. Port `_render_email`, `send_confirmation`, `send_reminder_24h`, `send_reminder_1h`, `build_manage_url`.
2. Subscribe to `appointment.created` event (no direct import of T12.7).
3. Replace `appointment_reminders` stub in `scheduler.py` with real scan for 24h+1h windows.
4. Include signed manage-URL (T12.10b) in each email.
5. Use T12.19 cooldown module to dedup against scheduler replays.
6. Log every send to `engagement_events` with appropriate category.
7. Render HTML-escaped first name, type, location, and city-local start time.

# Acceptance Criteria
