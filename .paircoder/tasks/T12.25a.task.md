# Objective

Port `ops:lib/nightly_reconcile.py` (`reconcile_bookings_sweep`, `advance_past_bookings`, `_should_advance`, `_write_audit_log`) to `backend/app/modules/appointments/reconcile.py`. If an appointment's `starts_at` is past and status is still `scheduled`, auto-mark `missed` after a 6h grace. Writes a one-time worker notification to `engagement_events(category='appointment_auto_missed_notice')` so the next daily digest surfaces an explicit "We marked this as missed" message with an "I actually attended" correction CTA — without this notice, workers would get stall reminders for a stall they didn't cause.

# Implementation Plan

1. Port `advance_past_bookings(grace_hours=6)` + helpers to sweep active sessions.
2. Mark past appointments beyond grace as `missed`.
3. Write audit row to `engagement_events(category='appointment_auto_advance', payload={...})`.
4. Write worker notice row to `engagement_events(category='appointment_auto_missed_notice')`.
5. Append outcome record via S12a T12.0a.
6. Wire into T12.25 orchestrator as step 2.5 (after retro, before digest composer).
7. Rely on S12a T12.18's 48h suppression window to prevent false-positive reminders.
8. Tests cover within-grace, beyond-grace, manual-attended, stall-suppression.
