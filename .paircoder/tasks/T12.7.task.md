# Objective

Create `backend/app/modules/appointments/scheduler.py` with pure CRUD: `create()`, `get()`, `list_by_session()`, `list_upcoming()`, `update()`, `mark_attended()`, `mark_missed()`, `cancel()`, `reschedule()`. Status transitions validated (port `ops:lib/job_tracker._check_status`). Event-driven coupling via `backend/app/core/events.py` lightweight in-process pub/sub; `appointment.created/attended/missed/rescheduled` events emitted. For S12a the only subscriber is an outcomes listener that appends to `outcomes_records`.

# Implementation Plan

1. Create `events.py` with `emit(event_name, payload)` + `subscribe(event_name, handler)`, thread-safe, sync dispatch.
2. Implement CRUD functions with status validation (port `_check_status`).
3. `mark_attended()` + `mark_missed()` emit events (never import consumers).
4. `reschedule()` preserves history in notes field.
5. Conflict detection: warn if new appointment overlaps existing for same session.
6. Create `outcomes_listener.py` that subscribes to all appointment events and writes to `outcomes_records` via T12.0a tracker.
7. Wire listener registration into `main.py` startup.
8. Tests: happy paths, all invalid transitions, event emission verified with test subscriber, outcome history durability.
