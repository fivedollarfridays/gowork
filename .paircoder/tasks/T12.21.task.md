# Objective

Create `backend/app/routes/engagement.py` with the four engagement endpoints not shipped in S12a (preview-digest is already live via S12a T12.21a). Endpoints: `GET /events`, `POST /preferences`, `POST /send-now` (admin, rate-limited), `POST /unsubscribe` (signed-token).

# Files to Update

| File | Change |

| backend/app/routes/engagement.py | create — 4 endpoints |
| backend/app/routes/__init__.py | register `engagement_router` |
| backend/tests/test_engagement_routes.py | create |

# Implementation Plan

1. `GET /events` — list engagement events for session.
2. `POST /preferences` — set `reminders_enabled`, digest schedule, etc.
3. `POST /send-now` — admin-only, rate-limited 3/hr per admin token; uses `require_admin_key` from `app.core.auth` (NOT a local constant).
4. `POST /unsubscribe` — verifies signed token via T12.10b pattern; sets `reminders_enabled=False`.
5. Tests cover auth boundaries (including send-now rate limit), each endpoint, unsubscribe round-trip.

# Acceptance Criteria

- [ ] Four endpoints implemented
- [ ] `send-now` uses `require_admin_key` from `app.core.auth` (NOT a local constant — do not follow `demo.py:20` pattern)
- [ ] `send-now` rate-limited (3 calls per admin token per hour)
- [ ] Unsubscribe endpoint verifies signed token (reuses T12.10b pattern) and sets `reminders_enabled=False`
- [ ] Tests cover auth boundaries (including send-now rate limit), each endpoint, unsubscribe round-trip
- [ ] `bpsai-pair arch check` passes
