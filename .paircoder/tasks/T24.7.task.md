# Objective

`POST /api/listings/{listing_id}/events {kind, session_id?, notes?}` records a reputation signal. Gated by `any_of_roles("case_manager", "admin")` — case managers record placements they witness; admins record manual events. Anonymous session_id permitted in payload so events tied to anonymous candidates aren't lost. Persists to listing_reputation_events with recorded_by = requesting account.

Lands in NEW route module — must NOT extend `auth.py`.
