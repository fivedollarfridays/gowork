# Admin feedback inbox + flagged-queue backend

New backend module + routes for the read-side admin surfaces over the existing S14 feedback substrate. No schema changes — `visit_feedback` already has the `reviewed INTEGER DEFAULT 0` and `action_taken TEXT` columns from m001. Two surfaces: flagged-resource queue (resources with `health_status='flagged'` joined with recent negative `resource_feedback` rows for context) and visit-feedback inbox (paginated browse with reviewed-state filter + mark-reviewed mutation). All routes admin role-gated. Pure SQLAlchemy `text()` + named binds (driver-agnostic; sqlite + postgres). Lives in NEW route module to avoid extending `routes/feedback.py` (which is candidate-facing). Router registration handled by T26.2 (single owner of `routes/__init__.py` edit this wave).
