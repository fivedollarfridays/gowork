# Objective

Port `ops:lib/booking_availability.py` (`compute_slots`, `compute_availability_range`, `_window_for_date`, `_busy_intervals_utc`, `_walk_slots`) + `ops:lib/booking_services.py` (`load_services`, `ServiceConfig`, `ServiceConfigError`). Service config is loaded from `cities/{city}.yaml` with per-type duration + hours. DST-aware, holiday-aware, and honors worker unavailability (T12.8a).

# Implementation Plan

1. Port `load_services` + `ServiceConfig` / `ServiceConfigError`; read from `cities/{city}.yaml`.
2. Port `compute_slots(location_hours, existing_appointments, slot_minutes, days_out)`.
3. Port `compute_availability_range(session_id, days_out)` returning per-day arrays.
4. Wire US federal holiday list (hardcoded initially) into `_window_for_date`.
5. Subtract worker unavailability blocks (T12.8a) from available slots.
6. DST-test across spring-forward + fall-back boundaries.
7. Write edge-case tests: fully-booked day, overnight, DST, holiday.
