# Objective

Port `ops:lib/booking_tokens.py` (`sign`, `verify`, `TokenInvalid`, `TokenExpired`, `TokenActionMismatch`) to `backend/app/modules/appointments/tokens.py`. HMAC-SHA256 signed tokens for email CTAs. Supports key rotation via `kid` payload field and atomic single-use enforcement via `used_tokens(token_hash UNIQUE)` to prevent double-click replay. Public `/api/appointments/manage` returns uniform 401 for all failure modes to block enumeration via error oracle.

# Implementation Plan

1. Port sign/verify + exception shapes; add `kid` field to payload.
2. Add m003 migration creating `used_tokens(token_hash TEXT UNIQUE, used_at TEXT)`.
3. Implement atomic single-use: `INSERT ... ON CONFLICT DO NOTHING`, then check rowcount.
4. Implement dual-secret verify: try current, fall back to `APPOINTMENT_TOKEN_SECRET_OLD` when `kid` matches.
5. Update `/api/appointments/manage` route to return identical 401 body for all failure modes.
6. Write rotation runbook `docs/ops/appointment-token-rotation.md`.
7. Tests cover sign/verify round-trip, expiry, action mismatch, concurrent-replay (threading×2), rotation, enumeration oracle.
