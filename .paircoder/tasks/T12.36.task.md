# Objective

Compliance gate for production GA. S12 added 13 sensitive tables (court dates, cleared-record status, barrier detail, resume content, email engagement). S12a's `ON DELETE CASCADE` handles session-level deletion but lacks worker-initiated export, selective delete, and retention beyond CASCADE. This task creates a compliance module + three user-facing endpoints + an internal retention sweep.

# Implementation Plan

1. Extend m003 migration with `compliance_audit` table + `deleted_at`/`deleted_reason` tombstone columns on sensitive tables.
2. Build async export: JSON + markdown archive of all worker rows across 13 new tables + `sessions`, `record_profiles`; emailed via signed T12.10b-pattern download link (24h TTL, single-use).
3. Build full-delete cascade with `compliance_audit` record per deletion.
4. Build selective-delete using tombstones; all reads filter `WHERE deleted_at IS NULL` for affected categories.
5. Build `retention_sweep()` purging sessions past `expires_at + 90d`; wire into T12.25 nightly orchestrator.
6. Require session-ownership token on all three user-facing endpoints.
7. Write runbook at `docs/ops/compliance-operations.md`.
8. Tests cover full export round-trip, full delete, selective delete visibility, retention sweep, signed-link expiry.

# Acceptance Criteria
