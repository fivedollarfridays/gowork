# Objective

Three pages under `/admin/assessments`: list of pending drafts (with track / kind / status filters), detail view of a single version (questions + review actions + admin-only publish button), and the typed API client. Styled with shadcn/Tailwind matching the existing GoWork palette (cyan/amber/rose). Hidden / 403-redirected if `useAccount()` returns null or the account lacks any reviewer role.

# Files

- `frontend/src/app/admin/assessments/page.tsx` (new — list)
- `frontend/src/app/admin/assessments/[versionId]/page.tsx` (new — detail)

# Acceptance Criteria

- [x] List page (`/admin/assessments`): fetches via `listPendingAssessments()`, shows pending drafts with filter dropdowns for track / kind / status; row click → detail
- [x] Detail page (`/admin/assessments/[versionId]`): fetches via `getAssessmentVersion`, shows questions ordered by position, comment textarea, action buttons (approve / reject / request_revision)
- [x] Admin-only publish button visible on detail page when status=approved (calls `publishAssessment(versionId)`)
- [x] `frontend/src/lib/api/assessments.ts` exposes typed: `listPendingAssessments()`, `getAssessmentVersion(versionId)`, `reviewAssessment(versionId, action, comment)`, `publishAssessment(versionId)`
- [x] Both pages 403-redirect to `/auth/login` if `useAccount()` is null
- [x] Both pages render "insufficient permissions" card if account lacks any reviewer role (uses `<RoleGate>` from T23.8 — but if T23.8 isn't merged yet, scaffold a local guard that T23.8 replaces)
- [x] Styling matches GoWork palette: action buttons use `bg-primary` (cyan) for approve, `bg-warning` (amber) for request_revision, `bg-destructive` (rose) for reject
- [x] vitest coverage: list rendering + filters, detail rendering + action submission, API client (request/response shapes + error states)
- [x] `npx tsc --noEmit` clean; `npx next build` green
