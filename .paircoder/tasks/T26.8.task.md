# Objective

Next.js page at `/admin/resources` with table view, filters, edit modal, hide/restore actions, and add-resource modal. Reuses the S23 `<RoleGate>` (inherited from `/admin/layout.tsx`). The shared `<ResourceForm>` component lives under `frontend/src/components/admin/` and is consumed by both add and edit modals.

Charter integrity: page does not import anything from `backend/app/modules/matching/`. T26.12 grep gate re-asserts.

# Files

- `frontend/src/app/admin/resources/page.tsx` (new)
- `frontend/src/components/admin/ResourceForm.tsx` (new — shared add/edit form)
- `frontend/src/app/admin/resources/__tests__/page.test.tsx` (new)

# Acceptance Criteria

- [ ] `frontend/src/app/admin/resources/page.tsx`:
  - Table with columns: name, category, city, health_status (badge), last edited (`user_curated_at` formatted via `Intl.DateTimeFormat`)
  - Filter dropdowns: city (FW, Dallas, Montgomery), category (from a static enum), health_status (healthy/watch/flagged/hidden — opt-in via "Show hidden" toggle)
  - Pagination controls: page-size 50, prev/next + page indicator
  - Row actions: Edit (opens modal with `<ResourceForm>` pre-filled), Hide (with confirmation dialog), Restore (only shown for hidden rows)
  - "Add Resource" button → modal with `<ResourceForm>` (empty state)
- [ ] `frontend/src/components/admin/ResourceForm.tsx`:
  - Fields: name, category, subcategory, address, lat (number), lng (number), phone, url, services (multi-select or comma-string), eligibility, city (dropdown)
  - Client-side validation: required = name, category, city; lat/lng must be numeric and within US-continental range (24-50 lat, -125 to -67 lng)
  - Uses shadcn `Input`, `Select`, `Textarea`
  - Submit handler accepts `(payload, mode: "create" | "edit", resourceId?)` — internally calls `createResource` or `updateResource` from T26.5
- [ ] `frontend/src/app/admin/resources/__tests__/page.test.tsx` covers:
  - Table renders with mocked API response (`listResources`)
  - Filters trigger refetch with correct params
  - Edit modal opens + populates from `getResource` + submits via `updateResource`
  - Hide action confirms via dialog + calls `hideResource`
  - Restore action calls `restoreResource`
  - Add modal validates required fields + lat/lng numeric range
- [ ] Page wrapped in `<RoleGate>` (already inherited from `/admin/layout.tsx`; explicit at page level optional)
- [ ] `npx tsc --noEmit` clean
- [ ] `npx next build` green
- [ ] No imports from `backend/app/modules/matching/` in the new page or shared component
